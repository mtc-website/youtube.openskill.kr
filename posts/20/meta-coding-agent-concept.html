00 · The Question

이 시리즈가 답하는 단 하나의 질문

"AI는 기억 없는 함수다. 그렇다면 어떻게 긴 일을 해내는가?"

이 질문 하나를 16편에 걸쳐 푼다. 모델의 본질(무상태 함수) → 그 한계(어텐션·rot) → 한계를 다루는 법(컨텍스트 엔지니어링) → 그래서 에이전트로 분리 → 자율 루프 → 신뢰성.

컨셉 한 줄 당신은 플레이어다. AI는 기억 없는 캐릭터다. 매 턴 '컨텍스트 보따리'를 통째로 건네야 그가 움직인다 — 그 보따리를 설계하는 법을 배운다.

01 · Metaphor

메타포 사전 — 모든 대본의 헌법

게임 척추는 스타크래프트 또는 포켓몬(미확정). 12·13편 디테일은 게임 확정 후 갱신, 사전·1강은 게임 무관하게 성립.

게임 요소개념주의
플레이어(나)main-agent / 오케스트레이터직접 전투 안 함, 무엇을 할지 결정·배치.
유닛/캐릭터sub-agent각자 독립된 시야(=컨텍스트 윈도우). 나누는 목적 = 컨텍스트 분리(주), 병렬(부).
캐릭터 고유 능력 Q/W/E/R역할/시스템 프롬프트⚠ 스킬 아님. 캐릭터 고정 정체성.
아이템 / 소환사 주문Skill (Agent Skill)누구나 장착·공유, 필요할 때 동적으로 꺼냄.

정찰·시야(fog of war / 파티 교체)컨텍스트 분리각 유닛은 자기 시야만 본다.

02 · Glossary

개념 사전 — 개념서의 색인
토큰 (Token)
모델이 글자가 아니라 잘게 쪼갠 단위. 컨텍스트·비용의 계량 단위.
임베딩 (Embedding)
의미를 좌표(벡터)로. 가까운 좌표 = 비슷한 의미.
무상태 (Stateless)
호출 간 기억 없음. 출력 = f(가중치, 입력 보따리).
컨텍스트 윈도우(보따리)
한 번의 호출에 들어가는 토큰 전체.
할루시네이션 / 작화
'참'이 아니라 '그럴듯함'을 출력해 생기는 자신 있는 오답.
어텐션 (Attention)
모든 토큰이 모든 토큰을 본다 → n² 비용.
KV 캐시
이전 토큰 K/V 저장해 재계산 제거. 대신 메모리가 새 병목.
Context rot
토큰이 늘수록 정확도가 완만히 떨어지는 현상.
컨텍스트 엔지니어링
매 추론에 어떤 토큰 집합을 넣을지 설계하는 일.
Compaction
보따리가 찰 때 요약 압축해 새 윈도우로 재시작.
Just-in-time 검색
참조만 두고 필요할 때 가져옴.
RAG
임베딩 검색으로 관련 문서를 미리 찾아 보따리에 넣음.
에이전트 (Agent)
도구를 루프에서 자율적으로 쓰는 LLM.
도구 / 함수 호출
모델이 바깥세계에 행동·조회하는 통로.
MCP
도구·데이터를 모델에 꽂는 표준 규격(USB-C 같은).
서브에이전트
깨끗한 별도 컨텍스트에서 깊게 일하고 요약만 반환.
Progressive disclosure
스킬을 메타데이터→본문→리소스 3단계 지연 로딩.
Orchestrator-worker
메인이 분해·합성, 워커가 독립 컨텍스트로 병렬 수행.
Eval
에이전트 성능을 재현 가능하게 측정하는 평가.

전체 커리큘럼 — 5막 16편
각 편: 메시지(한 줄) · 핵심(꼭 짚을 것) · 오해(흔한 착각 정정) · Sim(시뮬레이터) · 선수(선행 편).  ⓟ 핵심 스파인 ⓢ 심화(묶거나 생략 가능)
ACT 1

모델이라는 존재

— 무엇인가
1강AI는 세상을 토큰과 좌표로 본다토큰 & 임베딩ⓟ
모델은 글자가 아니라 토큰을 보고, 의미를 좌표(벡터)로 다룬다.
핵심토크나이즈(왜 "strawberry의 r 개수"를 틀리나) · 임베딩에서 가까움=비슷함 · 토큰이 비용·길이의 단위.
오해"AI가 글자를 읽는다" → 토큰 단위. "임베딩=DB" → 학습된 좌표공간.
Sim문장을 토큰으로 쪼개고, 단어를 2D 의미 지도에 배치.
선수없음(시작점).
2강이 캐릭터는 기억이 없다무상태 & 보따리ⓟ
AI는 매 턴 보따리를 통째로 다시 건네야 움직인다. "어제 대화"도 매번 다시 싼 것.
핵심출력 = f(고정 가중치, 보따리) · 왜 기억이 없어야만 하는가(아래 §기억 없는 이유) · 기억하는 건 모델이 아니라 앱(하네스).
오해"모델이 대화를 기억한다" → 앱이 DB에 저장해 매번 다시 넣는 것.
Sim빈 캐릭터에 보따리를 싸 던지면 행동 → 비우면 백지.
선수1강.
3강그럴듯함의 함정할루시네이션 & 샘플링ⓟ
모델은 '참'이 아니라 '그럴듯함'을 출력한다. "모른다"는 기본 상태가 없다.
핵심분포에서 샘플링(temperature/top-p, 온도↑→헛소리↑) · 검색이 아니라 생성 · 추측 보상(OpenAI 2025): 빈칸 0점·찍기 가끔 정답 → 찍게 학습됨.
오해"가장 높은 확률만 고른다" → 샘플링. "벡터에서 가까운 걸 찾는다" → 함수가 분포 생성. "환각" → 정확히는 작화(confabulation).
Simtemperature 0 vs 높음 / "모름" 허용 vs 차단 시 정답률·헛소리율.
선수1·2강.
ACT 2

컨텍스트의 물리학

— 한계
4강보따리는 공짜가 아니다어텐션 n² & KV 캐시ⓟ
토큰을 늘리려면 알고리즘이 필요하고, 늘려도 비싸진다.
핵심어텐션 n² 쌍 · 늘리는 알고리즘(FlashAttention, GQA/MQA, PagedAttention, RoPE 보간·YaRN) · KV 캐시는 재계산 제거하나 메모리가 새 병목 · prompt caching = 비용·지연만 해결.
오해"토큰 못 늘린다" → 늘리되 수확 체감. "캐시하면 길어도 괜찮다" → 품질은 별개.
Sim토큰↑ → 연결선 n² 폭증 + 비용 곡선, KV 캐시 on/off 비교.
선수1·2강.
5강보따리가 무거우면 둔해진다Context rot / lost in the middleⓟ
보따리가 길수록 모델은 멍청해진다. 특히 중간을 흘린다.
핵심시작·끝은 잘 보고 중간 누락 · Chroma 2025: 18개 모델 전부 저하 · 절벽 아닌 그래디언트.
오해"윈도우 안에만 있으면 다 본다" → 위치·길이에 따라 회수율 다름.
Simneedle 위치(앞/중간/끝)·길이별 회수 정확도 막대.
선수4강.
ACT 3

컨텍스트 엔지니어링

— 한계를 다루는 법
6강보따리 안에는 무엇이 들어가나컨텍스트 해부학ⓟ
좋은 보따리 = 결과 확률을 최대화하는 가장 작은 고신호 토큰 집합.
핵심6층(시스템 규칙·메모리·검색 문서·도구 스키마·최근 대화·현재 과제) · 시스템 프롬프트의 "적절한 고도" · 적은 수의 좋은 도구.
오해"많이 넣을수록 똑똑" → 신호 대 잡음이 중요.
Sim보따리 6칸을 채우며 신호/잡음 비율과 출력 품질 변화.
선수2·5강.
7강보따리 청소에는 두 개의 층이 있다컨텍스트의 'GC'ⓢ
컨텍스트 정리는 GC가 아니라 손실 요약/제거다.
핵심추론 레이어(KV eviction / StreamingLLM: sink+최근 윈도우) vs 앱 레이어(compaction, tool result clearing) · 자동 reachability 회수가 안 되므로 손실적.
오해"컨텍스트는 GC 못 한다" → 한다, 단 손실 요약 기반.
Sim보따리 항목이 분류되어 비워지거나 압축되는 과정.
선수5·6강.
8강버리지 말고 빼두기메모리 & 결정 로그 & negative knowledgeⓟ
중요한 건 버리지 말고 보따리 밖(파일)에 빼뒀다 다시 부른다.
핵심structured note-taking(NOTES.md/memory tool) · 결정 로그(ADR) · 실패한 시도와 이유(negative knowledge)도 일급 기록 · 안정적 프리픽스는 캐시도 잘 먹음.
오해"다 지우고 결정만" → 좋지만 실패 기록 없으면 같은 삽질 반복.
Sim결정·실패기록을 외부 노트로 빼고 다음 턴 다시 부르는 흐름.
선수7강.
9강미리 넣을까, 그때그때 가져올까RAG vs Just-in-timeⓟ
보따리를 사전에 채울지(RAG), 참조만 두고 필요할 때 가져올지(just-in-time)의 트레이드오프.
핵심RAG의 속도 vs just-in-time의 신선함·정확함 · 파일명·경로 같은 메타데이터가 신호 · 하이브리드(CLAUDE.md는 미리, grep는 그때그때).
오해"RAG가 항상 답" → 동적 작업엔 just-in-time이 나을 때가 많음.
Sim같은 질문을 RAG(미리) vs 탐색(그때그때)으로 처리, 속도·정확도 비교.
선수6강.
ACT 4

모델에서 에이전트로

— 자율
10강에이전트란 무엇인가 ★keystone루프 도는 LLMⓟ
에이전트 = 도구를 루프에서 자율적으로 쓰는 LLM. 워크플로(정해진 경로)와 다르다.
핵심관찰→생각→행동(도구)→관찰 루프 · 루프가 goal 달성까지 도는 사이 수많은 튜닝 · 모델이 똑똑할수록 자율↑.
오해"에이전트=챗봇" / "정해진 자동화 스크립트" → 자율 루프가 핵심.
SimLLM이 도구를 골라 호출→결과 보고 다음 행동 결정하는 루프 시각화.
선수2·6강.
11강도구와 MCP모델이 세계와 연결되는 법ⓟ
모델은 도구로만 바깥에 행동·조회한다. MCP는 그 도구를 꽂는 표준.
핵심함수 호출의 계약(이름·파라미터·반환) · 좋은 도구=적고·명확·토큰 효율 · MCP = "도구/데이터의 USB-C".
오해"모델이 직접 인터넷·DB를 본다" → 도구를 통해서만. "도구 많을수록 좋다" → 모호하면 해.
Sim같은 과제를 도구 없이 vs 도구로, MCP로 도구를 꽂는 그림.
선수10강.
12강왜 캐릭터를 나누는가서브에이전트 = 깨끗한 시야ⓟ
나누는 1번 목적은 병렬이 아니라 컨텍스트 분리.
핵심서브는 수만 토큰 탐색해도 1,000~2,000토큰 요약만 반환 → 메인 보따리 안 더러워짐 · 관심사 분리 · (fog of war / 파티 교체 = 시야 격리).
오해"여러 개 = 그냥 빨라지려고" → 1번은 시야 분리.
Sim메인 보따리는 깨끗, 서브 보따리만 부풀었다 요약만 귀환.
선수6·10강.
13강스킬은 아이템이다Agent Skill & progressive disclosureⓟ
스킬은 누구나 장착·공유하는 아이템. 필요할 때 동적으로 발견된다.
핵심3단계 지연 로딩(메타데이터 상주→SKILL.md→하위 리소스) · 범용 말고 프로젝트 맞춤 · 스킬이 소환수(sub-agent)를 부르기도.
오해"스킬은 미리 다 장착" → 동적 발견(컨텍스트 비대화 방지).
Sim아이템 상자에서 필요한 스킬만 꺼내 장착, 미사용 스킬은 보따리 밖.
선수11·12강.
14강부대 편성의 기술Orchestrator-worker & 병렬ⓟ
의존 없이 분해하면 병목이 사라진다. 여기서 비로소 병렬이 이득.
핵심오케스트레이터(분해·합성) + 워커(독립 컨텍스트 병렬) · 서로 안 기다리게 설계 · Anthropic: 복잡 작업 단일 대비 90%+ 향상.
오해"병렬=무조건 빠름" → 의존 있으면 병목·오염.
Sim직렬(한 줄 대기) vs 병렬(독립 분기) 처리 시간 비교.
선수12강.
ACT 5

신뢰성의 개념

— 개념만; 구현은 자매 시리즈
15강회귀와 피드백 루프통합테스트 게이트 & 실패의 재활용ⓟ
룰을 주입하기보다 테스트로 영향을 잡고, 깨지면 롤백 + 실패를 컨텍스트로 먹인다.
핵심"깨졌다"만이 아니라 "무엇을 왜 깼는가(영향 분석)"를 다음 시도 보따리에 · 빠른 유닛 루프 + 통합테스트 게이트 · 블라인드 재시도 = thrashing.
오해"TDD 시키면 회귀 막힘" → 절차만 강요하면 작은 모델은 회귀 증가(TDAD). 핵심은 테스트-컨텍스트.
Sim통합테스트 빨간불 → 롤백 → 실패 원인 카드가 다음 보따리에 삽입 → 재도전.
선수8·10강 (개념만. 실제 구현은 바이브 코딩 시리즈)
16강잘하는지 어떻게 아는가평가(Evals)ⓢ
에이전트는 "느낌"이 아니라 재현 가능한 평가로 검증한다.
핵심대표 예시 셋 · 좋은·나쁜 예 둘 다 · 결정성·회귀 추적 · 추측 보상 함정과 연결(평가가 행동을 만든다).
오해"잘 도는 것 같으면 됐다" → 평가 없으면 회귀를 모름.
Sim동일 과제 셋 점수판, 평가 기준 바꾸면 '최선'이 바뀌는 모습.
선수3·15강.

04 · Format

한 편 고정 포맷 (개념 전용, 10~13분)
  1. 0:00–1:00 훅 — 게임 상황으로 현상 제시("이 캐릭터가 왜 갑자기 멍청해졌을까?").
  2. 1:00–4:00 판서 — 원리 1개 손으로 전개(메타코딩 정체성).
  3. 4:00–8:00 시뮬레이터 — 그 개념 1개만 인터랙티브 (★ "개념 비유 — 실제 내부와 다름" 라벨 고정).
  4. 8:00–11:00 오개념 부수기 — 현실에서 이 개념을 몰라서 생기는 실패 1개(코딩 데모 아님, 사례·반례).
  5. 11:00–끝 — 1줄 요약 + 다음 편(다음 규칙/영입) 떡밥.
기존 "실전 한 스푼"(도구 데모)을 "오개념 부수기"로 교체 — 개념서답게 evergreen 유지, 도구 노후화 회피.

05 · Core Argument

2강 보강: "왜 기억이 없어야만 하는가"

현상만 가르치면 약하다. 네 겹의 이유로 "없어야만 한다"를 논증한다.

1 · 고정 함수학습 끝난 가중치는 얼어 있다. 출력 = f(가중치, 보따리). 대화해도 가중치는 안 바뀐다 → 저장 공간 자체가 없다.
2 · 확장성 (진짜 must)상태가 인스턴스에 남으면 다음 요청을 다른 GPU로 못 보낸다. 무상태라 어떤 복제본이든 보따리만 받으면 응대 → 수백만 동시 서비스(REST 무상태와 같은 이유).
3 · 재현·디버깅출력이 보따리에만 의존 → 같은 보따리=같은 행동. 숨은 기억이 있으면 추적 불가. "보따리에 없으면 일어나지 않은 일."
4 · 통제권상태가 밖에 명시적이라 편집·삭제·리셋 가능.
thesis 건망증은 극복할 결함이 아니라 당신에게 통제권을 주는 설계다. 얘가 아는 건 전부 당신이 보따리에 넣은 것뿐 — 그래서 플레이어가 중요하다.

06 · Length

길이 조절 가이드 (완전판 16편 ↔ 타이트 12편)

타이트하게 가려면 묶거나 줄인다.

  • 1강 임베딩 파트를 가볍게, 또는 3강과 통합.
  • 7강(GC) + 8강(빼두기)을 "보따리 관리" 한 편으로 병합.
  • 9강(RAG/just-in-time)을 6강(해부학) 뒤에 짧게 흡수.
  • 16강(평가)을 보너스/별도.
→ 핵심 스파인(ⓟ) 12편 + 심화(ⓢ)는 옵션. 실제 운영은 12편 스파인 + 심화편 별도 재생목록 추천.

07 · QA

사실 체크리스트 (대본 부착용)
  • "AI가 글자를 읽는다"고 안 했는가 → 토큰 단위.
  • "모델이 대화를 기억한다"고 안 했는가 → 앱이 매번 다시 넣는다.
  • 할루시네이션을 "검색 실패"로 안 했는가 → 그럴듯함 생성 + 추측 보상, 작화.
  • "가장 높은 확률"이라 단정 안 했는가 → 분포에서 샘플링.
  • "토큰 못 늘린다" → "늘리려면 알고리즘 + 수확 체감".
  • "캐시가 품질을 고친다"고 안 했는가 → 비용/지연만.
  • context rot을 "절벽"이라 안 했는가 → 그래디언트.
  • 컨텍스트 정리를 "GC"라 단정 안 했는가 → 손실 요약, 2층.
  • 결정 로그에 실패기록(negative knowledge) 포함했는가.
  • 에이전트를 "챗봇/정해진 스크립트"라 안 했는가 → 도구 쓰는 자율 루프.
  • "모델이 직접 인터넷·DB 본다"고 안 했는가 → 도구를 통해서만(MCP).
  • 서브에이전트 목적을 "컨텍스트 분리 1번, 병렬 보너스"로 말했는가.
  • 스킬을 캐릭터 고정이 아니라 아이템(공유·동적)으로 비유했는가.
  • TDD를 "절차"가 아니라 "테스트-컨텍스트 + 실패 피드백"으로 말했는가.
